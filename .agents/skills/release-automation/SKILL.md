## DetecciĂłn AutomĂˇtica de VersiĂłn (Semantic Versioning)

### Como determinar el tipo de bump:

#### 1. AnĂˇlisis de commits desde el Ăşltimo tag

#### 2. Conventional Commits (auto-detectar)
| Prefijo | Tipo de cambio | Bump |
|---------|----------------|------|
| `feat:` | Nueva funcionalidad | **MINOR** |
| `fix:` | Bug fix | **PATCH** |
| `chore:` | Mantenimiento | PATCH |
| `docs:` | DocumentaciĂłn | PATCH |
| `refactor:` | RefactorizaciĂłn | PATCH |
| `perf:` | Performance | PATCH |
| `test:` | Tests | PATCH |
| `BREAKING CHANGE:` | Cambio romper | **MAJOR** |
| `feat(breaking):` | Feature rompe | **MAJOR** |

#### 3. Detectar Breaking Changes
```bash
# Buscar BREAKING CHANGE en commits
echo "$COMMITS" | grep -i "BREAKING CHANGE\|breaking change\|breaking-change"

# Buscar commits con ! despuĂ©s del tipo
echo "$COMMITS" | grep -E "^[a-z]+!:"
```

#### 4. LĂłgica de versiĂłn automĂˇtica
```

SINO SI hay feat: (sin breaking) â†’ MINOR (x.Y.0)
```

## Flujo de Trabajo Completo

### 1. Pre-flight Checklist

### 2. AnĂˇlisis AutomĂˇtico de VersiĂłn
```bash

# Paso 2: Analizar commits

LAST_TAG=$(git describe --tags --abbrev=0 2>/dev/null || "v0.0.0")
COMMITS=$(git log ${LAST_TAG}..HEAD --oneline 2>/dev/null | wc -l)

if [ "$COMMITS" -eq 0 ]; then
  echo "No hay commits desde el Ăşltimo tag"
  exit 1
fi

# Paso 3: Detectar tipo de cambio

HAS_FEAT=$(git log ${LAST_TAG}..HEAD --oneline | grep -E "^feat:" || true)

if [ -n "$HAS_BREAKING" ]; then
  BUMP_TYPE="MAJOR"
elif [ -n "$HAS_FEAT" ]; then
  BUMP_TYPE="MINOR"
else
  BUMP_TYPE="PATCH"
fi
```

### 3. Calcular siguiente versiĂłn
```bash

MINOR=$(echo $CURRENT | cut -d. -f2)
PATCH=$(echo $CURRENT | cut -d. -f3)

case $BUMP_TYPE in
  MAJOR) NEW_VERSION="$((MAJOR+1)).0.0" ;;
  MINOR) NEW_VERSION="$MAJOR.$((MINOR+1)).0" ;;
  PATCH) NEW_VERSION="$MAJOR.$MINOR.$((PATCH+1))" ;;
esac
```

### 4. Git Operations
```bash
# Actualizar versiĂłn en Cargo.toml
sed -i "s/version = \"$CURRENT\"/version = \"$NEW_VERSION\"/" Cargo.toml

# Commit de release
git add Cargo.toml
git commit -m "Release v$NEW_VERSION"

# Crear tag anotado
git tag -a "v$NEW_VERSION" -m "Release v$NEW_VERSION

Changes:
- Type: $BUMP_TYPE
- Commits since last release: $COMMITS"

# Push
git push origin main
git push origin "v$NEW_VERSION"
```
